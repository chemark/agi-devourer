# 游戏 PRD 文档：《AGI 吞噬者》（原名：打嗝变色龙）

本文档包含项目的原始原型阶段记录（v1.0）以及正在规划中的商业化演进功能（v2.0）。

---

## 🟢 第一版 (Game Jam 原型版本 - v1.0)

### 1. 游戏概述
**标语 (Pitch)：** “你只能做一件事：按下空格，吞噬大模型，进化为真正的 AGI！”
**背景故事：** 一只背上印着 “AGI” 的终极赛博变色龙，想要突破算力奇点成为真正的数字神明。玩家需要控制它不断吐出变异的舌头，将天空中代表各大主流 AI 语言模型的“胶囊飞虫”吞入腹中进行融合。
**核心机制：** 单按键（空格/点击）街机反应游戏。极简的交互结合基于真实大模型（国内全域直连：MiniMax-M2.5）API 生成的动态反馈，给玩家带来充满荒诞科技圈黑话的无限“盲盒”喜剧体验。

### 2. 核心玩法 (Core Gameplay Loop)
1. **自动挂机瞄准（限制挑战）：** 变色龙的本能导致舌头瞄准虚线会像钟摆一样在上方（-45° 到 +45°）恒定来回匀速平移（速度 1.2，容错极高）。
2. **唯一交互输入（One Thing 原则）：** 全程玩家唯一的合法操作是：**看准时机，敲击空格键（或鼠标左键）**。按下后，舌头将沿着当前的瞄准轨道瞬间飞出。
3. **抓取与收网（宽幅碰撞）：** 将原本严格的坐标点重构成了“包含舌头红球半径的 +25px 宽幅擦边判定”。若击中了正在乱飞的大模型实体，会将其卷入口中，瞬间斩获其自带的基础算力值（例如 OpenAI: 300分，Kimi: 200分）。
4. **终极算力融合（大招释放）：** 累积吞噬满 **3** 个大模型后，胃部算力过载，强制接管并触发最核心的“大模型化学变异结算”。

### 3. API 叙事系统 (AI Innovation - 30% 权重)
作为占据评审 30% 创新比重的最大化抓手，我们彻底换下了必须依赖出海隧道的原始底层，直接切换接驳至具备超低无响应率的 **阿里云 DashScope (集成 MiniMax)**，通过标准的跨源格式进行稳定提取。
* **输入组合变量：** 传给 AI 的是这极短时间内被生吞的 3 家异构大语言模型的名字及其乱炖混战。
* **人格面具注入：** 极具幽默感的癫狂科技圈解说员。
* **输出结果：** 动态返回 1 句极限字数压缩的“算力碰撞事故小作文”（充满各种 IT 行业黑话或离谱的肉体赛博化变异），以及一个根据模型互斥程度给出的融合附加战力分（10~100）。
* **极端网络容灾设计：** 如果在内网路演或遭代理切断，触发捕获后系统将静默介入长达 5 套的随机“离线专属梗备用笑话库”（并且保底抽取 50~100 补充分数），保证游戏心流的绝对连贯，永不存在红字卡死问题。

### 4. 胜负体验规划 (Win & Lose Mechanism)
* **全局时限引擎：** 玩家拥有一段极短且无增量获取途径的 **120 秒** 倒计时压力阀。
* **赢者加冕：** 基础击杀分结合每次吃撑结算的 Gemini 爆分，一旦总额越过 **4000 算力临界区**，系统将瞬间强制锁定一切活动并高能宣布：“算力融合已达到临界值！变色龙突破奇点，进化为真正的 AGI！”
* **失败机制：** 如果手脚迟缓满场溜怪，致使 120 秒内依然在 4000 分大关外徘徊，将直接落入失败判定：“算力枯竭！在降智幻觉中停止了思考！”
* **无缝轮回的内存流机制：** 每局无论死活，均在 UI 高亮提供丝滑纯洁的纯逻辑数据劫持清空按钮“重新开始吞噬”，完成内存级变量零延迟清洗和开机。

### 5. 美术与音效 (Art & Audio)
* **2.5D 美术景深：** 高饱和全屏框架 (100vw/vh)。采用 AI 渲染完成的脱底天空辅以带有 CSS `filter` 对比亮色洗底魔法的底层变色龙模型，叠加极高文字排布占比的 AGI 渐变呼吸霓虹流光。将原来的靶形元素全面废除并进化为了胶囊宽横向条带展示的大语言模型的长文本牌匾。
* **纯代码轻巧声学结构：** 0 外挂 MP3 依赖。纯靠代码实例化浏览器的 AudioContext，榨取出最硬核无延时的 **8-bit** 频响器发声。含有：
    1. “嗖”：通过 `triangle` 三角波实现频率光速爬坡的弹射音。
    2. “吧唧”：由正弦波交叠形成的进食两极脆拍。
    3. “嗡——”：作为最终结算反馈音，基于底层锯齿波并附加癫狂的 `detune` 音高乱颤错位，硬生生合成出了“深腹空鸣”般荒加打颤的废料长嗝低音炮效果，完美契合魔幻叙事主题。

### 6. MVP 最终落地声明
全面秉承并且贯彻了在《评审维度.md》与《元提示词模板.md》中约定的那套理念：**斩除所有非核心代码边角料，去肥增瘦，只做验证闭环。**
本作抛弃了虚无缥缈的虚幻、Unity 以及任何超过 20M 的沉重外置引擎包，仅靠最基底、随时拖入任意现代浏览即可无阻通关的三个文本页面 `index.html` + `script.js` + `style.css`，直接在 6 小时的赛段内切中了题意的内核靶心，稳稳斩获“单点击交互”+“纯正 AI 化”的桂冠！

## 🚀 第二版 (商业化演进 - v2.0)

本阶段旨在将核心的“一键式搞怪吞噬”机制打磨并转化为具备用户黏性、社交裂变属性和富有硬核挑战梯度的完整线上产品。

### 1. 新增四大核心产品功能

#### 1.1 得分排行榜与用户注册
- 实现全网算力积分排行榜。
- 引入用户身份认证：支持游客模式极速畅玩，但在想要提交荣誉积分至全站榜单时，系统将强阻断并引导其进行注册与登入。
- 账号体系设计要求：打通社交关联，具备支持微信（扫码/网页静默鉴权）、手机号获取验证码及邮箱登录等多种主流通行规则。

#### 1.2 社交分享与裂变传播
- 在 Game Over 结算与突破奇点界面显著增加多渠道分享机制接口。
- 开发“朋友圈光荣战报”生成器，一键由前端抓取融合了“专属神级 AI 怪嗝文案”与得分的高清海报，并自动贴附游戏入口二维码，催化出站拉新。

#### 1.3 移动端深度适配优化
- 彻底摒弃仅仅支持 PC 端浏览器窗口游玩的环境，做出一套能完美支持在流媒体软件（如微信内）嵌框体验的视图。
- 视图重写支持响应式的极小屏尺寸，并从底层改造重修 Canvas 准心和触发反馈系统，让手机横竖屏下的单纯手指触控一样跟手丝滑。

#### 1.4 增加游戏难度 (变态进化曲线)
- 打破原有匀速护肝逻辑，引入时间维度的乘数引擎。随挂机时长递进，重力值和变色龙准心移动频率呈现倍率加速恶化，逼近人类极限阈值。
- 面临挑战目标时：给天际乱飞的这些大模型猎物灌注“防吞噬 AI 警觉神经元”。依据游玩进程推进，猎物轨道将演变出波浪线避震、急刹甚至是预判你的舌头打击而强行侧身规避的高级闪现诡迹。

### 2. 底层技术大变阵 (Architecture Refactoring)

为了托举住上述第二代的狂野蓝图，继续沿用上代纯裸 HTML 和单纯代理转发会立刻深陷泥潭，必须开展以下深水区级别换代：

#### 2.1 游戏引擎：Phaser.js
- **最终选定**：**Phaser.js** (基于 WebGL / HTML5 Canvas)。
- **决策理由**：在轻量化 2D 网页游戏中，它是王者，并且对于新手极其友好、中文教程极多。它能帮我们一键包揽未来要面对的“手机触屏事件映射”和“模型高难度变轨碰撞计算”的坑。相比之下 Godot 对本量级过重，PixiJS 偏向纯底层渲染，故 Phaser 是提效唯一选。

#### 2.2 前端界面框架：React
- **最终选定**：**React** (搭配 Vite 构建工具)。
- **决策理由**：React 依然拥有着全球最全的面包房（轮子库）。不论是我们需要引用的排行榜精美组件（如 Tailwind 系列），还是“在界面上将战绩弹窗直接转图并生成二维码海报”的开源包，React 生态都能直接白嫖。对于你的小白队友来说，写 React 组件远比手捏原生 DOM 节点可维护得多。

#### 2.3 后端数据库与 BaaS 服务：只选 Supabase
- **最终选定**：单挑 **Supabase**，全量托管。
- **避坑预警（绝不混用 Cloudflare D1）**：千万不要混用！Supabase 底层本身就是威力巨大的 PostgreSQL。对于只有双人甚至带新手的班子，如果试图在 Supabase 里做“用户认证（Auth）”，然后再跑去给“分数流（DB）”对接 Cloudflare 的 D1 边缘数据库，跨库的数据一致性逻辑会让你们生不如死。
- **正解**：全部押注交给 Supabase 一把梭，不仅自带免密邮箱/微信三方登录库，前端一句话就能拉下“Order By 得分排名前 100”的全服记录，完全砍掉后端接口研发！
