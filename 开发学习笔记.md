## 1. HTML5 Canvas 游戏开发基础

### 学到了什么

Canvas 是浏览器内绘图的核心 API，游戏的每一帧都是把画布清空再重新画。

```javascript
const canvas = document.getElementById('gameCanvas');
const ctx = canvas.getContext('2d');

function draw() {
    ctx.clearRect(0, 0, canvas.width, canvas.height); // 清空
    // ... 绘制所有元素
    requestAnimationFrame(draw); // 下一帧
}
```

### 常用绘图方法

| 方法 | 用途 |
|------|------|
| `ctx.beginPath()` | 开始一条新路径（必须在每次绘制前调用） |
| `ctx.fillStyle` | 设置填充颜色/渐变 |
| `ctx.strokeStyle` | 设置描边颜色 |
| `ctx.arc(x, y, r, 0, Math.PI*2)` | 画圆 |
| `ctx.ellipse(x, y, rx, ry, angle, 0, Math.PI*2)` | 画椭圆 |
| `ctx.fillText(text, x, y)` | 画文字 |
| `ctx.createLinearGradient()` | 线性渐变 |
| `ctx.createRadialGradient()` | 径向渐变（发光效果） |
| `ctx.save()` / `ctx.restore()` | 保存/恢复画布状态（防止样式污染） |

### 踩过的坑

**问题**：忘记 `ctx.save()` / `ctx.restore()`，一个元素设置的 `strokeStyle` 污染了后面所有元素的描边颜色。  
**解决**：每个独立的绘制块都包在 `ctx.save()` ... `ctx.restore()` 里。

## 2. requestAnimationFrame 游戏循环

### 学到了什么

`requestAnimationFrame` 是浏览器提供的「告诉我下一帧要画什么」接口，比 `setInterval` 更精准，自动适配屏幕刷新率（60fps / 120fps）。

```javascript
let lastTime = 0;

function gameLoop(timestamp) {
    const dt = (timestamp - lastTime) / 1000; // 转换成秒
    lastTime = timestamp;

    update(dt); // 用 dt 更新位置，让速度与帧率无关
    draw();

    requestAnimationFrame(gameLoop);
}

requestAnimationFrame(gameLoop);
```

### 关键：用 `dt`（delta time）而不是固定步长

- **错误做法**：每帧把物体移动 5px → 60fps 每秒移 300px，120fps 每秒移 600px，速度不一致
- **正确做法**：速度定义为「每秒移动多少像素」，每帧乘以 `dt`（上帧到这帧的秒数）

```javascript
// 速度 = 2500 px/s，不管帧率多少，每秒走的距离相同
tongue.length += tongue.speed * dt;
```

## 3. CSS Filter 颜色滤镜系统

### 学到了什么

这是本次开发中最有意思的技术之一——用纯 CSS 滤镜把图片染成任意颜色。

**公式**：
```css
filter: grayscale(1) sepia(1) hue-rotate(Xdeg) saturate(4) brightness(0.92);
```

**原理**：
1. `grayscale(1)` — 先把图片变成灰度（去掉原有颜色）
2. `sepia(1)` — 把灰度图染成暖棕黄色（色相约 30°）
3. `hue-rotate(Xdeg)` — 在 30° 基础上旋转 X 度，得到目标色
4. `saturate(4)` — 大幅提升饱和度，得到鲜亮颜色
5. `brightness(0.92)` — 微调亮度

**计算公式**：目标色相 = 30（sepia 基底） + hue-rotate 值

```javascript
// 各模型的 hue 值（hue-rotate 需要的度数）
{ text: 'OpenAI',  hue:  90 },  // 30+90=120°  翠绿
{ text: 'Claude',  hue:   5 },  // 30+5=35°    珊瑚橙
{ text: 'Gemini',  hue: 180 },  // 30+180=210° 宝石蓝

// 应用到变色龙
cEl.style.filter = `grayscale(1) sepia(1) hue-rotate(${hue}deg) saturate(4) brightness(0.92)`;
```

### 为什么不直接用 `hue-rotate`？

直接在原图上 `hue-rotate` 效果很差——原图本身就有多种颜色，旋转后每个部分都会往不同方向偏，整体颜色不统一。先 `grayscale` 归一化是关键。

## 4. BFS 洪水填充算法——浏览器内去白底

### 问题背景

变色龙图片是白底 PNG。用 `mix-blend-mode: multiply` 可以让白色"透明"，但一旦需要实色渲染（让变色龙变成指定颜色），就必须去掉 `multiply`——去掉后屏幕出现了一个巨大的白色矩形。

### 解决方案：离屏 Canvas + BFS

```javascript
async function removeBgFromImage(img) {
    // 1. 创建离屏 Canvas（不显示在页面上）
    const offCanvas = document.createElement('canvas');
    offCanvas.width = img.naturalWidth;
    offCanvas.height = img.naturalHeight;
    const offCtx = offCanvas.getContext('2d');
    offCtx.drawImage(img, 0, 0);

    // 2. 获取所有像素数据
    const imageData = offCtx.getImageData(0, 0, offCanvas.width, offCanvas.height);
    const data = imageData.data; // RGBA 数组，每4个元素是一个像素

    // 3. BFS：从四条边缘出发，把连通的白色像素变透明
    const visited = new Uint8Array(offCanvas.width * offCanvas.height);
    const queue = [];

    // 把四条边上的白色像素全部加入队列
    for (let x = 0; x < offCanvas.width; x++) {
        enqueueIfWhite(x, 0, ...);
        enqueueIfWhite(x, offCanvas.height - 1, ...);
    }
    for (let y = 0; y < offCanvas.height; y++) {
        enqueueIfWhite(0, y, ...);
        enqueueIfWhite(offCanvas.width - 1, y, ...);
    }

    // BFS 扩散
    while (queue.length > 0) {
        const [x, y] = queue.shift();
        const idx = (y * offCanvas.width + x) * 4;
        data[idx + 3] = 0; // 把 alpha 设为 0（透明）
        // 检查上下左右四个邻居
        [[x+1,y],[x-1,y],[x,y+1],[x,y-1]].forEach(([nx, ny]) => {
            if (isWhite(nx, ny) && !visited[...]) queue.push([nx, ny]);
        });
    }

    // 4. 把处理结果写回 Canvas，导出为 base64
    offCtx.putImageData(imageData, 0, 0);
    return offCanvas.toDataURL('image/png');
}
```

### 为什么用 BFS 而不是直接把所有白色变透明？

直接把所有白色变透明会破坏图片内部的白色细节（眼睛高光、皮肤反光等）。BFS 只消去**与边缘连通**的白色区域，内部的白色保留不变。

### 学到的知识点

- `getImageData()` 返回的 `data` 数组以 `[R, G, B, A, R, G, B, A, ...]` 格式排列，像素 `(x, y)` 的索引 = `(y * width + x) * 4`
- `toDataURL('image/png')` 把 Canvas 转成 base64 字符串，可以直接赋给 `<img src>`
- BFS（广度优先搜索）在图像处理中就是「洪水填充」（Flood Fill）的实现原理

## 5. Bezier 曲线绘制自然舌头

### 问题

用直线（梯形四边形）绘制舌头看起来很僵硬，像一根硬棍子。

### 解决

用 `quadraticCurveTo`（二次 Bezier 曲线）绘制舌体：

```javascript
const sway = Math.sin(performance.now() * 0.005) * (len * 0.04);
const midX = (bx + tx) / 2 + nx * sway; // 中间控制点横向偏移
const midY = (by + ty) / 2 + ny * sway;

ctx.beginPath();
ctx.moveTo(bx + nx * rootW, by + ny * rootW);          // 根部左边
ctx.quadraticCurveTo(midX + nx * midW, midY + ny * midW,
                     tx + nx * tipW, ty + ny * tipW);  // 到舌尖左边
ctx.lineTo(tx - nx * tipW, ty - ny * tipW);             // 舌尖右边
ctx.quadraticCurveTo(midX - nx * midW, midY - ny * midW,
                     bx - nx * rootW, by - ny * rootW); // 回到根部右边
ctx.closePath();
```

### 法线向量的作用

**法线** = 垂直于运动方向的向量，用来控制舌头的「厚度」：

```javascript
const ux = dx / len;  // 运动方向单位向量
const uy = dy / len;
const nx = -uy;       // 法线 = 旋转90度
const ny =  ux;

// 在法线方向上偏移 rootW 就是舌头的半宽
点A = (bx + nx * rootW, by + ny * rootW) // 根部一侧
点B = (bx - nx * rootW, by - ny * rootW) // 根部另一侧
```

## 6. CSS 动画与 mix-blend-mode 的坑

### 坑 1：`!important` 导致动画结束后样式无法恢复

**问题**：三连爆炸动画用 `!important` 强制设置了变色龙颜色，动画结束后想改回品牌色时，JS 设置的 `filter` 被 `!important` 覆盖，变色龙颜色卡住了。

**解决**：去掉 CSS 中的 `!important`，改用 JS 在 `animationend` 事件中直接设置 `style.filter`。

```javascript
chameleonEl.addEventListener('animationend', () => {
    chameleonEl.classList.remove('combo-burst');
    // animationend 后再设置，此时 CSS 已经不再竞争
    chameleonEl.style.filter = `grayscale(1) sepia(1) hue-rotate(${lastHue}deg) saturate(4)`;
}, { once: true });
```

### 坑 2：`mix-blend-mode: multiply` 与实色渲染不兼容

- `multiply`：把图片颜色与背景相乘，白色(#fff)与任何颜色相乘等于原色，即白色"消失"
- 问题：一旦需要显示实色（变色龙变颜色），`multiply` 会把纯色和背景混合，效果发灰
- 解决：用 BFS 在源头去掉白底，彻底移除 `mix-blend-mode`

## 8. Cloudflare Worker 作为 API 代理

### 为什么需要代理？

浏览器有**同源策略**（CORS）：前端 JS 不能直接请求第三方 API（如 api.deepseek.com），除非对方设置了 `Access-Control-Allow-Origin: *`。

解决方案：自己部署一个代理服务器，给浏览器的请求加上 CORS 头再转发。

### Worker 核心代码

```javascript
export default {
    async fetch(request, env) {
        // 处理预检请求
        if (request.method === 'OPTIONS') {
            return new Response(null, {
                status: 204,
                headers: {
                    'Access-Control-Allow-Origin': '*',
                    'Access-Control-Allow-Methods': 'POST, OPTIONS',
                    'Access-Control-Allow-Headers': 'Content-Type, Authorization',
                }
            });
        }

        const body = await request.text();
        const res = await fetch('https://api.deepseek.com/chat/completions', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'Authorization': `Bearer ${env.DEEPSEEK_API_KEY}`, // 从环境变量读
            },
            body,
        });

        return new Response(await res.text(), {
            status: res.status,
            headers: { 'Content-Type': 'application/json', 'Access-Control-Allow-Origin': '*' }
        });
    }
}
```

### 部署命令

```bash
# 安装 CLI
npm install -g wrangler

# 登录 Cloudflare
wrangler login

# 部署 Worker
wrangler deploy

# 把 API Key 存为环境变量（不写在代码里！）
```

## 10. 云彩 CSS 随机动画系统

### 用 JS 生成随机 CSS 动画

不需要图片，纯 CSS `border-radius` + `filter: blur` 就能做出逼真云朵：

```javascript
function createCloud() {
    const cloud = document.createElement('div');
    const size = 80 + Math.random() * 120;
    cloud.style.cssText = `
        position: absolute;
        width: ${size * 2}px;
        height: ${size}px;
        background: rgba(255,255,255,${0.15 + Math.random() * 0.2});
        border-radius: 50%;
        filter: blur(${8 + Math.random() * 12}px);
        top: ${Math.random() * 50}%;
        left: ${Math.random() * 100}%;
        animation: cloudDrift ${20 + Math.random() * 30}s linear infinite;
        animation-delay: ${-Math.random() * 30}s; /* 负延迟让动画错开 */
    `;
    return cloud;
}
```

### `animation-delay` 负值的技巧

设置负的 `animation-delay`（如 `-15s`）相当于让动画提前开始 15 秒——页面加载时云朵就已经在运动中了，不会有「所有云同时从同一位置出发」的问题。

## 11. 游戏状态机设计

### 学到了什么

复杂游戏用「状态机」管理，避免状态混乱。

```javascript
// 游戏只有这几种状态，互斥
let gameState = 'start';     // 开始界面
// gameState = 'playing'     // 游戏进行中
// gameState = 'api_result'  // 三连结果展示（可继续）
// gameState = 'gameover'    // 游戏结束

function update(dt) {
    if (gameState !== 'playing') return; // 非游戏中直接跳过
    // ...
}

// 每个事件处理前先判断当前状态
window.addEventListener('keydown', (e) => {
    if (e.code === 'Space') {
        if (gameState === 'playing') {
            shootTongue();
        } else if (gameState === 'api_result') {
            resumeGame();
        }
    }
});
```

### 踩过的坑

**问题**：游戏结束后（gameover），计时器 `setInterval` 还在跑，`timeLeft` 继续递减到负数。  
**解决**：在 `setInterval` 回调最开始加 `if (gameState !== 'playing') return;` 守门。

## 12. API Key 安全实践

### ❌ 错误做法（千万不要）

```javascript
// script.js（前端代码，所有人都能看到）
const API_KEY = 'sk-****************************'; // ❌ 暴露！永远不要这样做
```

任何人打开浏览器开发者工具 → Sources，就能看到这个 Key，然后用你的 Key 随意调 API。

### ✅ 正确做法：Key 存在后端环境变量

```bash
# 通过 CLI 把 Key 存到 Cloudflare Worker 的加密存储里
echo "sk-yourkey" | wrangler secret put DEEPSEEK_API_KEY
```

```javascript
// Worker 代码里通过 env 对象读取，永远不会暴露给前端
'Authorization': `Bearer ${env.DEEPSEEK_API_KEY}`
```

前端只知道 Worker 的 URL，不知道 Key 是什么。

## 13. Cloudflare Pages 与 Worker 的分工与关系（FAQ）

### Q: 这个游戏项目到底是用 Cloudflare 的 Page 还是用它的 Worker 来部署的？我搞不懂



在这个游戏项目中，其实是**把 Pages 和 Worker 结合起来一起使用的**，它们有各自明确的分工：

#### 1. Cloudflare Pages：负责「前端（游戏显示）」
- **作用**：负责托管和部署你的 HTML、CSS 和前端 JavaScript 代码。
- **机制**：玩家访问的诸如 `agi-tongue-game.pages.dev` 就是 Pages 部署的网页。它直接发给玩家浏览器进行渲染，玩家实际看到和操作的游戏画面，全部运行在这里。

#### 2. Cloudflare Workers：负责「后端（API 代理跑腿）」
- **作用**：作为一个安全的后端服务节点运行。
- **机制**：为了防止将大模型（如 DeepSeek）的 API Key 直接明文写在前端代码里被人盗用，前端代码会拦截这个流程，反而去向你的 Worker 发送请求（如 `agi-tongue-proxy...workers.dev`）。Worker 被配置了安全保管箱（环境变量），它会带着存放在里面的安全密钥向大模型发起调用，完成后把回复发还给前端。

**总结**：玩家访问的是 **Pages**（你的主游戏）。当触发 AI 生成对话时，前端的 JS 偷偷呼叫 **Worker** 帮忙。Worker 充当安全中介请求大模型，这保证了访问速度极快，同时也死死守住了你的 API 资产安全。

## 🔖 其他零散知识点

### Canvas 碰撞检测

圆形碰撞检测（两圆心距离 < 半径之和）：

```javascript
function checkCollision(px, py, circle) {
    const dx = px - circle.x;
    const dy = py - circle.y;
    const dist2 = dx * dx + dy * dy; // 用平方避免开方（性能优化）
    return dist2 <= (circle.size + hitRadius) ** 2;
}
```

### `performance.now()` vs `Date.now()`

- `Date.now()`：毫秒级精度，受系统时间影响
- `performance.now()`：微秒级精度，从页面加载开始计时，更适合动画

### `[...new Set(array)]` 数组去重

```javascript
const uniqueModels = [...new Set(eatenModels)]; // 简洁去重
```

### CSS `pointer-events: none`

让元素不接受鼠标事件（点击穿透），适合覆盖在游戏上的 UI 层：

```css
#ui-layer { pointer-events: none; }
```
