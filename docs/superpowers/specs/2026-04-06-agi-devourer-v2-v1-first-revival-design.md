# 《舌尖上的 AGI》V2 V1-First 复活版设计文档

日期：2026-04-06
状态：已通过脑暴确认，待用户审阅书面 spec
范围：将当前 V2 从“产品壳优先、局内简化占位”调整为“V1 局内完整复活，V2 产品能力轻量外挂”

## 1. 设计摘要

新版 V2 不再继续沿着当前这套简化 Canvas 局内推进。

新的方向是：

1. 直接复活 V1 的完整局内体验
2. 保留 V2 已经明确正确的产品闭环
3. 让 V1 负责游戏核心，让 V2 负责外层登录、榜单和结果承接

这次改版的中心目标不是“把现有页面美化一下”，而是修正两个根本问题：

- 当前启动流程会让手机用户产生“点了开始但没反应”的误解
- 当前局内视觉、手感和表达力都明显低于 V1，不足以支撑项目的游戏身份

新版 V2 的标准是：

- 看起来先像一个真正的游戏，再像一个轻产品化活动页
- 点击开始后体验顺畅，不再有多一步确认造成的误导
- 局内恢复 V1 的美术、操作、音效和胜负结构
- 结算后仍然能走 V2 的“游客先玩，登录上榜”链路

## 2. 核心方向

### 2.1 总体定位

V2 的正确形态是：

一个以 V1 局内为核心、以 V2 结果转化为外层闭环的移动端 H5 游戏。

优先级排序如下：

1. 游戏是否重新拥有 V1 级别的存在感
2. 开局是否顺畅，不再误导
3. 结果页是否能自然引导登录与上榜
4. 首页是否像游戏启动页，而不是产品说明页

### 2.2 产品边界

这次明确采用以下职责拆分：

- V1 局内负责“玩得爽”
- 启动页负责“让人想点开始”
- 结果页负责“让游客愿意登录上榜”
- 登录负责“确认身份并补交本局成绩”
- 榜单负责“给成绩社会反馈”

局内不再承担登录、榜单、表单和过重产品提示。

## 3. 启动流与页面结构

新版 V2 采用 4 层结构，但视觉上尽量表现为同一套游戏世界，而不是多个产品页拼接。

### 3.1 游戏启动页

首页必须从“产品卡片页”改成“游戏启动页”。

它的职责只有：

- 建立世界观和情绪
- 展示主角或主视觉
- 给出一个非常明确的“开始挑战”
- 轻量挂载登录状态和榜单入口

首页不再承担：

- 大段产品说明
- 大块榜单卡片展示
- 强引导先登录

### 3.2 横屏校验覆盖层

点击“开始挑战”后的逻辑改为：

- 如果设备已经横屏，直接进入游戏
- 如果设备仍是竖屏，不再跳到单独的“横屏准备页”
- 改为在启动页上方出现一个强提示的横屏覆盖层
- 用户把设备横过来后，自动进入局内，不需要再点第二次开始

这是本轮必须修掉的关键交互问题。

### 3.3 V1 Phaser 局内

局内恢复为 V1 风格的横屏 Phaser 场景：

- 占满主要横屏视野
- 保留背景、变色龙本体、飞行目标和舌头发射
- 保留 V1 的节奏、反馈和挑战结构

### 3.4 结果页 / 结果层

一局结束后进入结果层：

- 先承接 V1 的胜负情绪和战报感
- 再承接 V2 的登录和上榜逻辑
- 游客可以在这里完成登录并把本局成绩送上榜
- 已登录用户可以直接获得排名反馈并再来一局

## 4. 首页视觉与信息架构

### 4.1 首页身份

首页必须被重新定义为“游戏启动页”，而不是“产品首页”。

用户第一眼应该感受到的是：

- 这是一个有角色感和情绪的小游戏
- 我现在可以马上开始

而不是：

- 这是一个有很多功能入口的活动页

### 4.2 首页必备内容

首页只保留以下内容：

- 游戏标题《舌尖上的 AGI》
- 一句极短的挑衅式副标题
- 主按钮“开始挑战”
- 一个轻量榜单入口或 Top 3 摘要
- 一个轻量登录状态提示

### 4.3 首页明确不要的内容

首页不再出现：

- 大块“游客先玩，登录后上榜”的说明文
- 过重的榜单卡片
- 类似 SaaS / 后台风格的玻璃卡片布局
- 多块功能面板纵向堆叠

### 4.4 视觉气质

首页视觉方向要向 V1 的游戏感靠拢：

- 更强的背景氛围
- 更有角色感的主视觉
- 更接近游戏 logo 的标题表达
- 更像开场页而不是功能页

V2 产品入口存在，但全部退居二线。

## 5. V1 局内复活范围

### 5.1 第一批必须恢复的内容

这次不是“参考 V1”，而是明确复活以下 V1 要素：

- Phaser 局内框架
- 背景素材
- 变色龙主角本体
- 更有辨识度的飞行目标
- 摆动瞄准
- 舌头发射
- 音效与 BGM
- 60 秒挑战
- 目标分机制
- 胜负态和结束情绪

### 5.2 可以逐步补齐的内容

以下内容重要，但不要求第一步做到完全体：

- 连击表现强化
- 更完整的吞噬反馈文案
- 飞行目标行为差异细化
- 更强的结算战报包装

### 5.3 明确不再继续的路线

这次不再继续投入以下方向：

- 继续给当前极简 Canvas 局内打补丁
- 在玩法没有救回之前继续加强产品外壳
- 发明一套与 V1 明显不同的新局内机制

## 6. 局内规则策略

### 6.1 完整沿用 V1 局内规则

局内规则优先采用“完整沿用 V1”：

- 单局 60 秒
- 有明确目标分
- 有胜负态
- 有连击与战报感

V2 不把局内改成单纯的“60 秒刷分演示”，而是保持 V1 的街机挑战结构。

### 6.2 原则

这轮的原则是：

- 局内越接近 V1 越好
- 外层只做轻量产品化，不反向破坏局内

## 7. V2 产品能力挂载方式

### 7.1 保留的 V2 能力

以下 V2 能力保留：

- 游客先玩
- 结算后登录上榜
- mock 微信登录
- 榜单能力
- 结果页回流

### 7.2 各层挂载位置

产品能力的挂载方式如下：

- 首页：只显示轻量登录状态和榜单入口
- 局内：不出现登录表单和榜单面板
- 结果页：承接登录、上榜、再来一局
- 登录成功后：回到当前结果，并自动补交刚才这局成绩

### 7.3 保持 mock 边界

第一轮继续沿用 V2 现有 mock 服务边界：

- `authService`
- `leaderboardService`
- `reportService`

这次不优先接真微信授权，也不优先接真后端。

## 8. 技术策略

### 8.1 总体策略

技术上采用“V1 局内复活，V2 外壳承接”的方式，而不是整体推翻重做。

### 8.2 局内

局内尽量直接复用旧分支 `codex/chameleon-v2` 中更完整的 Phaser 能力与素材：

- `GameScene`
- `PhaserGame`
- 素材与音频资源
- 旧版局内控制和状态表达

### 8.3 外层壳

外层继续保留当前 V2 的 React 状态流与 mock 服务结构，但会调整以下部分：

- 首页改成游戏启动页
- 删除单独“横屏准备页”
- 增加横屏覆盖层逻辑
- 结果页视觉和承接方式向 V1 情绪靠拢

### 8.4 结果桥接

V1 Phaser 局内结束后，必须把以下结果数据桥接给 V2 外层：

- 分数
- 是否达成目标分
- 剩余时间或局时信息
- 吞噬结果摘要
- 可用于结算文案和战报展示的基础数据

## 9. 实施顺序

本轮按以下顺序实施：

1. 修启动流  
   完成“开始挑战 -> 横屏覆盖 -> 自动进局”，彻底移除第二次点击开始的误导。

2. 替换局内  
   移除当前简化 Canvas 占位局内，接回 V1 Phaser 局内和素材。

3. 修结果承接  
   让 V1 局内结束后进入 V2 结果页，并带出胜负态、分数和结果摘要。

4. 重做首页  
   把首页改成真正的游戏启动页，并只保留极轻的登录状态和榜单入口。

## 10. 验收标准

这一轮完成后，至少要满足以下标准：

1. 手机用户点击开始时，不再出现“像没反应”的体验
2. 设备横屏后可以自动进入局内
3. 局内明显恢复到接近 V1 的视觉和手感
4. 仍然保留 V2 的“游客先玩，结算后登录上榜”闭环
5. 首页已经具有明确的游戏启动感，而不是产品后台感

## 11. 非目标

这一轮明确不做：

1. 真微信 OAuth
2. 真后端排行榜
3. 首页重产品化堆叠
4. 新发明一套替代 V1 的局内机制
5. 在当前极简局内上继续追加视觉补丁

## 12. 结论

V2 的正确方向不是继续强化当前这套“产品链路正确，但游戏本身太弱”的结构。

正确方向是：

- 用 V1 救回游戏核心
- 用 V2 保留产品闭环
- 让启动页更像游戏开场
- 让结果页承担登录和上榜转化

这是一条以游戏感优先、产品能力后挂的 V2 复活路线。
