# 🤝 舌尖上的 AGI - 队友协作新手指南

欢迎加入《舌尖上的 AGI》项目！

考虑到你可能对代码版本控制系统（Git 和 GitHub）还不太熟悉，这份指南将用最简单的大白话和最少的操作，带你完成从“拿到代码”到“把你的修改发布到全网”的完整流程。

我们已经配置好了前台页面的**全自动部署（CI/CD）**，所以你**只要把代码改好交上去，过 2 分钟全世界就能看到你的更新**。

请跟着下面的两步走：

---

## 🛠️ 第一步：把代码搞到自己电脑上（只做一次）

作为新手，**极其不建议你去背那些复杂的黑框命令行**。我们强烈推荐使用官方提供的可视化软件。

1. **下载工具并登录**：
   - 下载并安装 [GitHub Desktop](https://desktop.github.com/)（完全免费的安全软件）。
   - 打开后，用你的 GitHub 账号登录。
2. **克隆（Clone）项目**：
   - 找队长要到本项目在 GitHub 上的地址链接。
   - 在 GitHub Desktop 软件中点击左上角 `File` -> `Clone repository...`。
   - 切换到 `URL` 选项卡，把项目链接贴进去，选一个你电脑上的文件夹来保存，点击蓝色的 `Clone` 按钮。

🎉 **恭喜**！现在游戏的全部源代码都在你电脑里了。

---

## 🎮 第二步：开辟专属工作区（创建你的分支）

为了防止大家同时改文件发生“撞车”，也为了保护线上稳定版本不被害死，我们采用**“分支协作流”**。每做一个新功能，都请为自己拉起一条独立的时间线：
- 在 GitHub Desktop 的正上方找到 **`Current branch`**（当前分支：通常默认是 main），点击它。
- 点击 **`New branch`** 创建新分支。
- 跟着任务榜单随便起个名字，比如做移动端适配就叫 `feature/mobile-ui`。
- 现在，你拥有了一个完全隔绝、怎么改都不会影响主干的虚拟沙盒！

---

## 💻 第三步：日常改代码、发私服（日常循环）

### 1. 本地修改与测试
- 进入你刚下载的分支文件夹的 `chameleon` 目录，**用浏览器直接双击打开 `index.html`**。
- 尽情改代码（换图、改 CSS 等），改完按 `Ctrl+S` 保存，切回浏览器按 `F5` 刷新立刻看效果。一切都在纯本地发生，放心大胆改！

### 2. 存入本地进度（Commit）
- 打开 GitHub Desktop，左边会自动列出你刚才改过的所有文件。
- 在左下方的 `Summary` 输入框里，**用一句直白的话写下你的功劳**（例如：“把舌头改红了”）。
- 点击底部的大蓝按钮 **`Commit to feature/...`** 存个本地进度。

### 3. 将你的进度送上测试服体验（Push）
- 点击软件顶部画着小云朵和箭头的按钮：**`Publish branch`** 或是 **`Push origin`**。
- 等进度条走完。
🚀 **完成！去爽一把你的专属内测服：**
由于我们装载了最先进的自动构建化机器人，Cloudflare 会监测到你的新分支，并且**为你这个分支单独跑起了一个测试网址**（比如 `mobile-ui.xxxx.pages.dev`）。用这个私密网址发给朋友测试你的新功能，爽就完事了！

---

## 🤝 第四步：测试通过，完美交接给主站（提 PR）

当你在你的分支测试环境里觉得功能已经极其完美，准备给正式网友玩的时候：
1. 不要用任何复杂的代码命令！直接去 GitHub 网页端的该项目页面。
2. 网页会自动弹出一个绿色提示，问你是不是刚发了新分支，点击 **`Compare & pull request`**。
3. 写两句话描述你做了什么牛逼的功能，提交这个请求。
4. 队长审核没大问题后，在线点一下 **`Merge pull request`**。

此时，你的心血才算真正并入了天下大同的 `main` 主干道，线上几分钟后自动刷新，全世界的玩家都能玩到你的成就了！

---

## 🆘 终极防坑急救箱

- **如果按 `Push origin` 报错了咋办？**
  这是因为在你改代码的时候，队长也偷偷在这个空当交了新代码，你们的进度“分叉”了。
  解决办法：点击界面上方的 **`Fetch origin`**，然后再点一下出现的 **`Pull origin`**（把队长的代码拉取下来跟你合并）。只要没有大冲突，拉完之后你就能顺利点 `Push` 把自己的送上去了！
- **我改了一半觉得完全是一团糟，想重来怎么办？**
  在左侧改动的文件列表上，**右键你要放弃的文件**，选择 **`Discard changes...`**。这会让文件像时光倒流一样瞬间恢复到你改动前的原始干净状态。
